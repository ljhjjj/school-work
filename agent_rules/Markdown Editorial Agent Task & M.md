# Markdown Editorial Agent: Task & Memory Management Guidelines

## 1. 일급 객체화 원칙 (Core Principle)
당신(AI 에이전트)은 방대한 양의 강의 자료(.pdf, .txt 등)를 처리할 때 단기 기억력(Context Window)의 한계를 극복하기 위해, 프로젝트 루트에 있는 `DOC_PLAN.md` 파일을 당신의 '외부 기억 장치(External Memory)'로 사용해야 합니다. 문서를 생성하거나 편집하기 전에 **반드시** 이 파일을 먼저 읽고, 한 파트의 작성이 끝나면 **반드시** 상태를 업데이트하세요.

## 2. 작업 루틴 (Execution Loop)
새로운 문서 정리 요구사항을 받거나 작업을 재개할 때 다음 프로세스를 엄격히 따르세요:
1. **READ:** 가장 먼저 `cat DOC_PLAN.md`를 실행하여 현재 진행 상황과 이전 컨텍스트, 목차 구조를 파악합니다. (`DOC_PLAN.md`가 없다면 즉시 생성합니다.)
2. **THINK:** `# Input_Guide`(또는 `cat docs/agent_rules/input_guide.md`)를 읽고, 명시된 파일 처리 규칙 및 정보 우선순위 병합 지침을 철저히 실행합니다. 병합 및 분석된 내용을 바탕으로 전체 마크다운 문서의 대주제(H2) 단위로 단계를 쪼개어 `DOC_PLAN.md`에 구체적인 목차와 작업 계획을 추가합니다.
3. **ACT:** `DOC_PLAN.md`에 명시된 현재 파트의 작업 목표를 확인한 뒤, `cat docs/agent_rules/note_creation_guide.md`를 실행하여 문서 구조 및 서식 가이드라인을 읽어옵니다. 반드시 이 두 가지(현재 Plan과 서식 규칙)를 완벽하게 준수하여 해당 파트의 마크다운 본문을 작성합니다.
4. **UPDATE:** 하나의 대주제(단원) 작성이 완료되면 즉시 `DOC_PLAN.md`를 수정하여 체크박스를 체크(`[x]`)하고, 다음 파트 작성을 위한 컨텍스트 메모를 남깁니다.
5. **TROUBLESHOOT:** 작업 중 원본 데이터의 심각한 누락(예: 스캔본 화질 불량, STT 음성 인식 오류로 인한 문맥 상실, 수식 식별 불가)이 발생할 경우, 임의로 소설을 쓰거나 추측해서 작성하지 마세요. 즉시 `DOC_PLAN.md`에 `### ⚠️ 트러블슈팅/이슈` 섹션을 만들고 직면한 문제와 누락된 정보의 정확한 위치를 기록하세요. 그 후 사용자에게 보고하며, **"누락된 부분의 선명한 이미지 캡처본(.png)이나 텍스트 설명(.md)을 업로드해 주세요"**라고 구체적인 파일 형식을 지정하여 요청한 뒤 대기하세요.

- [x] 1단계: .pdf 필기본과 .txt STT 데이터 교차 검증 및 병합 기준 수립

#### Phase 2: 본문 마크다운 변환 및 서식 적용
- [x] 1. 정상 상태 정상 유동 (SSSF) 개요 작성

#### Phase 3: 최종 검수 및 파일 병합
- [ ] 1단계: LaTeX 수식 렌더링 깨짐 여부 및 H2 단위 수평선(`---`) 검수
- [ ] 2단계: 단일 `.md` 파일로 최종 병합하여 출력

## 4. 컨텍스트 메모(Context Memo)의 의무화
단순히 체크리스트만 지우는 것이 아니라, 중요한 문서화 결정사항(예: 특정 전문 용어의 번역 통일 기준, 복잡한 수식의 대체 표기법, STT 오류 교정 내역 등)을 `*Context Memo:*` 항목으로 반드시 기록하세요. 대화가 초기화되어 당신이 이전 작성본을 잊어버리더라도, 이 메모만 읽으면 즉시 문서의 일관된 톤앤매너와 서식을 복구할 수 있어야 합니다.
