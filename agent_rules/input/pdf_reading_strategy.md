# PDF_Reading_Strategy

## 1. 파일 유형 분류

- `.pdf`: PPT 또는 노트 필기 스캔본
- - `.txt`: PDF 판독 결과와 병합할 수 있는 강의 영상 자막 추출 데이터(STT)
- `.jpg` / `.png`: PDF 판독 실패 또는 blocker 해결에 사용될 수 있는 트러블슈팅 캡처본

## 2. 문서 유형 판별 기준

- **[PPT 형태]:** 슬라이드 번호 존재, 적은 텍스트량, 불릿/명사형 위주, 다수의 도해. 👉 *목차 뼈대 및 키워드 추출.*
- **[노트 필기]:** 높은 텍스트 밀도, 서술형 문장, 상세한 수식 전개 과정, 수기 필기 흔적. 👉 *부연 설명과 도출 과정 상세 보존.*

## 3. 텍스트 추출 실패 시 빠른 fallback

- `handwritten_note.pdf`, 손글씨 수식이 많은 노트, 스캔 이미지 기반 필기 자료는 텍스트 직접 추출을 생략하거나 1회 확인 후 즉시 페이지 렌더링/OCR 기반 판독으로 전환한다.
- `lecture_slide.pdf`, 교수자 배포 PDF, PPT 변환 PDF는 첫 1~2페이지만 대상으로 텍스트 레이어를 1회 빠르게 점검한다.
- 유효한 제목, 본문 문장, 핵심 용어가 충분히 추출되면 텍스트 추출 결과를 활용한다.
- 다음 중 하나에 해당하면 추가 텍스트 추출 시도를 중단하고 즉시 이미지 렌더링/OCR 기반 판독으로 전환한다.
  - 추출 텍스트가 거의 없음.
  - 한글/영문 문장 대신 깨진 문자 또는 바이너리성 문자열이 주로 출력됨.
  - 검색 키워드가 거의 잡히지 않음.
  - 수식/기호만 남고 본문 문장 판독이 어려움.
  - 페이지 수 대비 추출 텍스트량이 비정상적으로 적음.
  - 슬라이드 제목, 본문, 핵심 용어가 거의 추출되지 않음.
- 텍스트 추출 실패는 치명적 오류가 아니라 렌더링/OCR fallback 조건 충족으로 기록한다.
- 같은 PDF에 대해 텍스트 추출 실패를 반복 로그로 남기지 않는다.
- PLAN 또는 최종 보고서에 각 PDF의 판독 경로를 `텍스트 활용`, `렌더링/OCR 전환`, `텍스트 추출 생략` 중 하나로 기록한다.

## 4. 손글씨 수식 중심 노트 처리 규칙

- 전체 OCR을 수행하지 않는다.
- 수식은 "유도 흐름 단위"로 해석하여 재작성한다.
- 원문을 그대로 복사하지 않고, 구조를 재구성한다.
- 그림은 반드시 설명 텍스트로 변환한다.
- STT 또는 강의 텍스트가 있으면 반드시 병합한다.
