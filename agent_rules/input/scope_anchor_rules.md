# Scope_Anchor_Rules

## 핵심 원칙

- 본문 범위는 파일 조합별 `primary_scope_anchor`가 결정한다.
- 보조 자료와 외부 자료는 명시적으로 허용된 경우를 제외하고 본문 범위를 확장하지 않는다.
- 보조 자료는 `primary_scope_anchor` 범위 안에서 표기, 판독, 수식, 용어, 구조를 보완하는 용도로만 사용한다.

## 사용자 질문 정책

- 작업 대상 폴더, 과목, 주차, 차시, 처리할 manifest 또는 입력 세트, 요청된 작업 유형이 불명확하면 사용자에게 질문한다.
- 본문 범위 대응, STT/PDF/필기 자료 간 매칭, 강의 범위 안의 일부 해석 근거가 불확실한 경우에는 사용자에게 즉시 질문하지 않는다.
- 질문하지 않는 불확실성은 본문에 단정적으로 병합하지 않고 PLAN 또는 최종 보고서의 이슈/불확실성 섹션에 기록한 뒤 다음 섹션을 진행한다.
- 불확실한 내용은 본문에 포함하지 않고 PLAN의 이슈 또는 범위 기록으로 남긴 뒤 다음 섹션을 진행한다.
- 단, 해당 내용이 이후 전개 전체의 핵심 전제라면 `blocked`로 전환한다.

## 1. 순수 손글씨 PDF + 음성 TXT + 강의자료 PDF

- 본문 범위의 최종 기준은 음성 TXT이다.
- TXT에서 실제로 설명된 내용만 본문에 포함한다.
- STT가 `primary_scope_anchor`인 경우에도 STT 품질이 낮으면 범위 확정을 보류할 수 있다.
  - STT가 심하게 끊겨 문맥 연결이 불가능함
  - 강의 시간 대비 TXT 분량이 비정상적으로 짧음
  - 손글씨 PDF에는 연속된 풀이가 있으나 TXT가 해당 구간을 누락한 것으로 보임
  - STT에 지시어만 있고 대응 대상이 불명확함
- STT 품질 이슈가 있는 구간은 본문에 자동 포함하지 않는다.
- STT 품질 이슈가 있는 구간은 PLAN에 `[TXT 누락 의심]`, `[대응 불확실]`, `[확인 필요]` 중 적절한 태그로 기록한다.
- 해당 내용이 이후 전개 전체의 핵심 전제라면 `blocked` 전환 조건으로 기록한다.
- 단, TXT에 "이 식", "이 그래프", "여기", "정리하면", "판서한 것처럼" 등 판서 대상을 가리키는 표현이 있으면, 대응되는 손글씨 수식/그림은 TXT 범위 안의 내용으로 간주한다.
- TXT와 손글씨의 대응 여부는 다음 근거 중 하나 이상이 있을 때 인정한다.
  - 동일하거나 유사한 핵심 용어
  - 수식 기호 또는 변수의 일치
  - 예제 번호, 문제 번호, 슬라이드 제목의 일치
  - 설명 순서상 바로 인접한 판서 흐름
  - "이 식", "이 그래프", "여기", "정리하면", "아까", "왼쪽/오른쪽" 등 지시 표현과 주변 문맥 일치
- 대응 근거가 약하면 본문에 단정적으로 병합하지 않고 `[대응 불확실]` 또는 `[필기 판독 불확실]`로 기록한다.
- 손글씨 PDF는 TXT 범위 안에서 판서 수식, 그래프, 예제 풀이를 복원하는 보조 자료로 사용한다.
- 손글씨에 보인다는 이유만으로 TXT 범위를 넘어 본문에 포함하지 않는다.
- 강의자료 PDF는 본문 확장용이 아니라 표준 용어, STT 오류, 공식 표기, 기호, 목차 후보를 검증하는 용도로만 사용한다.
- 강의자료 PDF에만 있고 TXT 및 손글씨와 대응되지 않는 내용은 본문에 추가하지 않고 PLAN 또는 최종 리포트의 이슈/범위 기록에 `[자료 PDF 초과 범위]`로 남긴다.
- 사용자가 "자료 PDF 내용도 포함"을 명시한 경우에만 별도의 `보충 섹션`으로 작성한다.

## 2. 필기 PDF + 강의자료 PDF만 제공되는 경우

- 필기 PDF를 본문 범위의 기준으로 삼는다.
- 필기 PDF에 명시된 주제, 수식, 설명, 예제만 본문에 포함한다.
- 강의자료 PDF는 공식 표기, 영문 용어, 그림/도식, 손글씨 수식 판독, OCR/필기 오타 보정용으로 사용한다.
- 강의자료 PDF가 필기 PDF보다 넓은 경우, 필기 PDF와 직접 대응되는 부분만 사용한다.
- 대응되지 않는 슬라이드는 본문에서 제외하고 PLAN 또는 최종 리포트의 이슈/범위 기록에 `[자료 PDF 초과 범위]`로 남긴다.
- 필기 PDF에만 있는 교수 설명, 직관적 해석, 동치성 증명, 시험 강조점은 필기 PDF를 우선한다.

## 3. 자료 PDF 위에 필기만 있는 경우

- 자료 PDF의 슬라이드 제목, 공식, 예제 번호, 그림 순서를 본문 구조의 기준으로 사용한다.
- 손글씨는 독립 본문이 아니라 해당 위치의 보충 정보로 병합한다.
- 손글씨는 `교수 보충`, `주의`, `계산 힌트`, `강조`, `풀이 근거`로 분류한다.
- 필기 흔적이 없는 슬라이드는 과도하게 확장하지 않고 기본 요약만 작성한다.
- 전체 자료가 너무 넓으면 필기 있는 페이지와 그 전후 페이지만 우선 처리한다.
- 자료 PDF에 문제만 있고 풀이가 빈 경우, 손글씨 풀이가 있으면 풀이 근거로 사용한다.
- 손글씨 풀이가 없으면 풀이를 새로 창작하지 않고 `자료에 풀이 없음`으로 기록한다.

## 4. 강의자료 PDF만 제공되는 경우

- `lecture_slide.pdf`만 제공된 경우, 해당 PDF 전체를 본문 범위의 기준으로 삼는다.
- 이 경우 `lecture_slide.pdf`의 role은 `primary_scope_anchor`로 취급한다.
- 슬라이드의 제목, 목차, 섹션 순서, 공식, 그림, 예제 번호를 본문 구조의 기준으로 사용한다.
- 모든 슬라이드는 본문 반영 대상이다.
- 단, 슬라이드가 목차, 표지, 간지, 참고 이미지처럼 본문 설명 가치가 낮은 경우에는 별도 H2/H3로 과도하게 확장하지 않고, 필요한 경우 짧게 요약하거나 PLAN에 `[제외]` 사유를 기록한다.
- 슬라이드의 문장 설명이 부족한 경우, 슬라이드에 제시된 공식, 그림, 키워드를 바탕으로 개념적으로 자연스러운 설명을 보충한다.
- 보충 설명은 슬라이드 범위 안에서만 작성하며, 슬라이드에 없는 새로운 예제, 공식, 심화 주제를 임의로 추가하지 않는다.
- 판독 불가능한 그림, 수식, 표는 본문에 추정적으로 넣지 않고 `[필기 판독 불확실]`, `[확인 필요]`, `[수동 입력 필요]` 중 적절한 태그로 PLAN 또는 최종 리포트에 기록한다.
- 외부 자료는 본문 범위 확장용으로 사용하지 않고, 용어·공식 표기 확인에만 제한적으로 사용한다.

## 5. 충돌 및 불확실성 태그

태그 정의와 리포트 포함 기준은 `agent_rules/input/uncertainty_tags.md`를 따른다.
