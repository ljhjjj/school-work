# Input_Guide

## [P2] 1. 파일 유형 및 분류
- `.txt`: 강의 영상 자막 추출 데이터(STT)
- `.pdf`: PPT 또는 노트 필기 스캔본
- `.jpg` / `.png`: 트러블슈팅 캡처본

## [P1] 2. AI 행동 지침 (Data Processing & Workflow)
**[작업 범위 제한]: 새로운 강의 노트 생성/수정 작업에서 파일을 입력받거나 작업을 시작할 때** 다음 지침에 따라 처리하세요. (단순 질문 시 무시)

1. **정보 읽기 및 우선순위 병합 (Preprocessing)**
   - **[판독 신뢰도 기준]** PDF/TXT 간 표기 충돌을 판독할 때만 일반적으로 `노트 필기 PDF > PPT 형태 PDF > 자막 추출 TXT` 순으로 표기 정확도를 높게 본다.
   - 단, 이는 본문 범위 결정 우선순위가 아니다. 본문에 어떤 내용을 포함할지는 파일 조합별 `primary_scope_anchor`가 결정한다.
   - **[.txt 처리: 신뢰도 기반 3단계 STT 교정 원칙]:**
     - **[Tier 1] 확정:** PDF, STT, 문맥이 모두 일치하거나 표준 용어로서 대체 후보가 사실상 하나뿐인 경우. 👉 *본문 즉시 반영.*
     - **[Tier 2] 추정:** 문맥상 가능성이 높으나, 대체 가능한 유사 후보가 2개 이상인 경우. 👉 *가장 적합한 단어로 작성하되 최종 리포트에 추정 내역을 기록.*
     - **[Tier 3] 보류:** 판독 불가. 👉 *원문 유지 후 `[?]` 표시하고 이슈로 기록.*
   - **[.pdf 처리: 명확한 문서 유형 판별 기준]:**
     - **[PPT 형태]:** 슬라이드 번호 존재, 적은 텍스트량, 불릿/명사형 위주, 다수의 도해. 👉 *목차 뼈대 및 키워드 추출.*
     - **[노트 필기]:** 높은 텍스트 밀도, 서술형 문장, 상세한 수식 전개 과정, 수기 필기 흔적. 👉 *부연 설명과 도출 과정 상세 보존.*
   - **[PDF 판독 전략: 텍스트 추출 실패 시 빠른 fallback]:**
     - `handwritten_note.pdf`, 손글씨 수식이 많은 노트, 스캔 이미지 기반 필기 자료는 텍스트 직접 추출을 생략하거나 1회 확인 후 즉시 페이지 렌더링/OCR 기반 판독으로 전환한다.
     - `lecture_slide.pdf`, 교수자 배포 PDF, PPT 변환 PDF는 첫 1~2페이지만 대상으로 텍스트 레이어를 1회 빠르게 점검한다.
     - 유효한 제목, 본문 문장, 핵심 용어가 충분히 추출되면 텍스트 추출 결과를 활용한다.
     - 다음 중 하나에 해당하면 추가 텍스트 추출 시도를 중단하고 즉시 이미지 렌더링/OCR 기반 판독으로 전환한다.
       - 추출 텍스트가 거의 없음.
       - 한글/영문 문장 대신 깨진 문자 또는 바이너리성 문자열이 주로 출력됨.
       - 검색 키워드가 거의 잡히지 않음.
       - 수식/기호만 남고 본문 문장 판독이 어려움.
       - 페이지 수 대비 추출 텍스트량이 비정상적으로 적음.
       - 슬라이드 제목, 본문, 핵심 용어가 거의 추출되지 않음.
     - 텍스트 추출 실패는 치명적 오류가 아니라 렌더링/OCR fallback 조건 충족으로 기록한다.
     - 같은 PDF에 대해 텍스트 추출 실패를 반복 로그로 남기지 않는다.
     - PLAN 또는 최종 보고서에 각 PDF의 판독 경로를 `텍스트 활용`, `렌더링/OCR 전환`, `텍스트 추출 생략` 중 하나로 기록한다.
   - **[손글씨 수식 중심 노트 처리 규칙]:**
     - 전체 OCR을 수행하지 않는다.
     - 수식은 "유도 흐름 단위"로 해석하여 재작성한다.
     - 원문을 그대로 복사하지 않고, 구조를 재구성한다.
     - 그림은 반드시 설명 텍스트로 변환한다.
     - STT 또는 강의 텍스트가 있으면 반드시 병합한다.

### [P0] 파일 조합별 본문 범위 및 검증 우선순위(primary_scope_anchor)

#### 사용자 질문 정책

- 작업 대상 폴더, 과목, 주차, 차시, 처리할 manifest 또는 입력 세트, 요청된 작업 유형이 불명확하면 사용자에게 질문한다.
- 본문 범위 대응, STT/PDF/필기 자료 간 매칭, 강의 범위 안의 일부 해석 근거가 불확실한 경우에는 사용자에게 즉시 질문하지 않는다.
- 질문하지 않는 불확실성은 본문에 단정적으로 병합하지 않고 PLAN 또는 최종 보고서의 이슈/불확실성 섹션에 기록한 뒤 다음 섹션을 진행한다.

- 불확실한 내용은 본문에 포함하지 않고 PLAN의 이슈 또는 범위 기록으로 남긴 뒤 다음 섹션을 진행한다.
- 단, 해당 내용이 이후 전개 전체의 핵심 전제라면 `blocked`로 전환한다.
#### 1. 순수 손글씨 PDF + 음성 TXT + 강의자료 PDF

- 본문 범위의 최종 기준은 음성 TXT이다.
- TXT에서 실제로 설명된 내용만 본문에 포함한다.
- STT가 `primary_scope_anchor`인 경우에도 STT 품질이 낮으면 범위 확정을 보류할 수 있다.
  - STT가 심하게 끊겨 문맥 연결이 불가능함
  - 강의 시간 대비 TXT 분량이 비정상적으로 짧음
  - 손글씨 PDF에는 연속된 풀이가 있으나 TXT가 해당 구간을 누락한 것으로 보임
  - STT에 지시어만 있고 대응 대상이 불명확함
- STT 품질 이슈가 있는 구간은 본문에 자동 포함하지 않는다.
- STT 품질 이슈가 있는 구간은 PLAN에 `[TXT 누락 의심]`, `[대응 불확실]`, `[확인 필요]` 중 적절한 태그로 기록한다.
- 해당 내용이 이후 전개 전체의 핵심 전제라면 `blocked` 전환 조건으로 기록한다.
- 단, TXT에 "이 식", "이 그래프", "여기", "정리하면", "판서한 것처럼" 등 판서 대상을 가리키는 표현이 있으면, 대응되는 손글씨 수식/그림은 TXT 범위 안의 내용으로 간주한다.
- TXT와 손글씨의 대응 여부는 다음 근거 중 하나 이상이 있을 때 인정한다.
  - 동일하거나 유사한 핵심 용어
  - 수식 기호 또는 변수의 일치
  - 예제 번호, 문제 번호, 슬라이드 제목의 일치
  - 설명 순서상 바로 인접한 판서 흐름
  - "이 식", "이 그래프", "여기", "정리하면", "아까", "왼쪽/오른쪽" 등 지시 표현과 주변 문맥 일치
- 대응 근거가 약하면 본문에 단정적으로 병합하지 않고 `[대응 불확실]` 또는 `[필기 판독 불확실]`로 기록한다.
- 손글씨 PDF는 TXT 범위 안에서 판서 수식, 그래프, 예제 풀이를 복원하는 보조 자료로 사용한다.
- 손글씨에 보인다는 이유만으로 TXT 범위를 넘어 본문에 포함하지 않는다.
- 강의자료 PDF는 본문 확장용이 아니라 표준 용어, STT 오류, 공식 표기, 기호, 목차 후보를 검증하는 용도로만 사용한다.
- 강의자료 PDF에만 있고 TXT 및 손글씨와 대응되지 않는 내용은 본문에 추가하지 않고 PLAN 또는 최종 리포트의 이슈/범위 기록에 `[자료 PDF 초과 범위]`로 남긴다.
- 사용자가 "자료 PDF 내용도 포함"을 명시한 경우에만 별도의 `보충 섹션`으로 작성한다.

#### 2. 필기 PDF + 강의자료 PDF만 제공되는 경우

- 필기 PDF를 본문 범위의 기준으로 삼는다.
- 필기 PDF에 명시된 주제, 수식, 설명, 예제만 본문에 포함한다.
- 강의자료 PDF는 공식 표기, 영문 용어, 그림/도식, 손글씨 수식 판독, OCR/필기 오타 보정용으로 사용한다.
- 강의자료 PDF가 필기 PDF보다 넓은 경우, 필기 PDF와 직접 대응되는 부분만 사용한다.
- 대응되지 않는 슬라이드는 본문에서 제외하고 PLAN 또는 최종 리포트의 이슈/범위 기록에 `[자료 PDF 초과 범위]`로 남긴다.
- 필기 PDF에만 있는 교수 설명, 직관적 해석, 동치성 증명, 시험 강조점은 필기 PDF를 우선한다.

#### 3. 자료 PDF 위에 필기만 있는 경우

- 자료 PDF의 슬라이드 제목, 공식, 예제 번호, 그림 순서를 본문 구조의 기준으로 사용한다.
- 손글씨는 독립 본문이 아니라 해당 위치의 보충 정보로 병합한다.
- 손글씨는 `교수 보충`, `주의`, `계산 힌트`, `강조`, `풀이 근거`로 분류한다.
- 필기 흔적이 없는 슬라이드는 과도하게 확장하지 않고 기본 요약만 작성한다.
- 전체 자료가 너무 넓으면 필기 있는 페이지와 그 전후 페이지만 우선 처리한다.
- 자료 PDF에 문제만 있고 풀이가 빈 경우, 손글씨 풀이가 있으면 풀이 근거로 사용한다.
- 손글씨 풀이가 없으면 풀이를 새로 창작하지 않고 `자료에 풀이 없음`으로 기록한다.

#### 4. 강의자료 PDF만 제공되는 경우

- `lecture_slide.pdf`만 제공된 경우, 해당 PDF 전체를 본문 범위의 기준으로 삼는다.
- 이 경우 `lecture_slide.pdf`의 role은 `primary_scope_anchor`로 취급한다.
- 슬라이드의 제목, 목차, 섹션 순서, 공식, 그림, 예제 번호를 본문 구조의 기준으로 사용한다.
- 모든 슬라이드는 본문 반영 대상이다.
- 단, 슬라이드가 목차, 표지, 간지, 참고 이미지처럼 본문 설명 가치가 낮은 경우에는 별도 H2/H3로 과도하게 확장하지 않고, 필요한 경우 짧게 요약하거나 PLAN에 `[제외]` 사유를 기록한다.
- 슬라이드의 문장 설명이 부족한 경우, 슬라이드에 제시된 공식, 그림, 키워드를 바탕으로 개념적으로 자연스러운 설명을 보충한다.
- 보충 설명은 슬라이드 범위 안에서만 작성하며, 슬라이드에 없는 새로운 예제, 공식, 심화 주제를 임의로 추가하지 않는다.
- 판독 불가능한 그림, 수식, 표는 본문에 추정적으로 넣지 않고 `[필기 판독 불확실]`, `[확인 필요]`, `[수동 입력 필요]` 중 적절한 태그로 PLAN 또는 최종 리포트에 기록한다.
- 외부 자료는 본문 범위 확장용으로 사용하지 않고, 용어·공식 표기 확인에만 제한적으로 사용한다.


#### 5. 충돌 및 불확실성 태그

정상 판정/범위 관리 태그는 validation에서 무조건 warning으로 잡지 않는다. 잔여 이슈 태그만 최종 리포트의 `[잔여 이슈]`에 포함한다.

##### 정상 판정/범위 관리 태그

- `[확정]`: 두 개 이상의 원본 자료가 일치함.
- `[보정]`: 필기/OCR/STT가 불명확하지만 강의자료로 확인됨.
- `[필기 우선]`: 강의자료에는 없지만 필기 PDF에 명확히 있음.
- `[TXT 우선]`: 강의자료에는 있으나 TXT에서 설명되지 않아 제외함.
- `[자료 PDF 초과 범위]`: 강의자료에는 있으나 현재 본문 범위를 넘어감.
  - 모든 슬라이드나 페이지를 개별 기록하지 않는다.
  - 페이지 범위, 섹션명, 대표 주제 단위로 요약 기록한다.
  - 예: `[자료 PDF 초과 범위] 18~25쪽: 피로 파괴 심화 내용, TXT/필기 대응 없음`
  - 초과 범위 내용은 본문에 추가하지 않고 PLAN 또는 최종 리포트의 이슈/범위 기록에만 남긴다.
- `[제외]`: 현재 입력 조합의 범위 기준을 벗어나 본문에 포함하지 않음.

##### 잔여 이슈 태그

- `[?]`: 판독 불가 또는 확정 불가.
- `[필기 판독 불확실]`: 손글씨가 작거나 흐려 확정할 수 없음.
- `[TXT 누락 의심]`: STT 품질이 낮거나 TXT가 손글씨 PDF의 연속 풀이 구간을 누락한 것으로 보여 범위 확정을 보류함.
- `[대응 불확실]`: TXT와 손글씨의 대응 근거가 약해 본문에 단정적으로 병합하지 않음.
- `[확인 필요]`: 자동 확정이 어려워 사용자 또는 후속 검증 확인이 필요함.
- `[수동 입력 필요]`: 자동 판독이 불가능해 사용자의 직접 입력이 필요함.

2. **DOC_PLAN 설계 및 작성 (Planning)**
   - 본문 작성 전, **해당 차시의 계획 파일(`output/[과목명]/[과목명]_[주차]-[차시]_PLAN.md`)**에 단계별 작업 계획을 먼저 작성하세요.
   - PDF 판독 전략을 정한 경우 PLAN에 `## 입력 판정` 섹션을 두고 다음 형식으로 기록하세요.
     - `handwritten_note.pdf`는 손글씨 문서로 판단되어 텍스트 직접 추출을 생략하고 페이지 렌더링/OCR로 판독함.
     - `lecture_slide.pdf`는 텍스트 레이어를 1회 점검했으나 판독 효율이 낮아 페이지 렌더링/OCR로 전환함.
     - 텍스트 추출 실패는 오류가 아니라 렌더링/OCR fallback 조건 충족으로 기록함.
     - 판독 경로: `handwritten_note.pdf` = `텍스트 추출 생략`, `lecture_slide.pdf` = `렌더링/OCR 전환`.
     - 렌더링 방식: 페이지별 렌더링을 사용함. Edge screenshot은 기본 판독 경로로 사용하지 않음.
     - 렌더링 캐시: 기본 모드에서는 작업 종료 후 삭제함. 디버그 모드에서는 `output/_render_cache/[과목명]/[주차]-[차시]/`에 보관함.
   - `lecture_slide.pdf`만 제공된 경우 PLAN의 `## 입력 판정` 섹션에 다음 내용을 기록하세요.
     - `lecture_slide.pdf`만 제공되어 해당 PDF 전체를 본문 범위 기준(`primary_scope_anchor`)으로 사용함.
     - 텍스트 레이어를 1회 점검하고, 유효하면 텍스트 추출 결과를 활용함.
     - 텍스트 추출이 불충분하면 페이지별 렌더링/OCR로 전환함.
     - 판독 경로: `lecture_slide.pdf` = `텍스트 활용` 또는 `렌더링/OCR 전환`.
     - 외부 자료는 본문 범위 확장용으로 사용하지 않고 용어·공식 표기 확인용으로만 사용함.
   - PDF 또는 이미지 입력을 판독한 경우, Full PLAN에서는 PLAN에 `## 페이지 판독 기록` 섹션을 두고 페이지별 근거를 남긴다.
     - Mini PLAN에서는 페이지 판독 기록은 선택 사항이다. 필요한 경우에만 `p.1~5: 핵심 개념 확인` 형태로 1줄 요약한다.
     - Full PLAN에서는 페이지 단위 판독 기록을 권장하되, 연속 구간은 묶어 기록한다.
     - 각 항목은 `파일명 p.번호: 핵심 개념, 공식, 도식, 용어, 사용 목적` 형식으로 기록한다.
     - 손글씨/필기 PDF는 본문 반영 근거인지, 판독 불확실 항목인지 구분한다.
     - 슬라이드 PDF는 본문 확장용인지, 용어·공식 보조 확인용인지 구분한다.
     - Full PLAN에서는 `lecture_slide.pdf`만 제공된 경우 모든 페이지를 판독 기록 대상으로 삼고, 페이지가 많으면 연속 구간 단위로 묶어 기록할 수 있다.
     - 예: `handwritten_note.pdf p.1: 제2법칙, 열효율, COP, 열저장조, Kelvin-Planck, Clausius`
     - 예: `handwritten_note.pdf p.2: Kelvin-Planck statement, Clausius statement, 두 명제의 동치성 논증`
     - 예: `handwritten_note.pdf p.3: 도식 논증, reversible/reversed process 구분 및 반영 필요 항목`
     - 예: `lecture_slide.pdf p.10~12: 열저장조, 열효율, COP 용어·공식 보조 확인`
     - 예: `lecture_slide.pdf p.1~3: 강의 제목, 일정, 전체 목차 확인`
     - 예: `lecture_slide.pdf p.4~7: 핵심 개념과 주요 예제 확인`
     - 예: `lecture_slide.pdf p.8~12: 공식, 그림, 계산 흐름 확인`
     - 예: `lecture_slide.pdf p.13~18: 적용 개념, 다이어그램, 예제 문제 확인`

3. **이미지 파일(`.jpg`, `.png`) 트러블슈팅 처리 규칙 (Resolving Blockers)**
   - 타겟 PLAN 파일의 `### ⚠️ 트러블슈팅/이슈` 섹션을 읽고 문제를 판독합니다.

   - **[상태 업데이트 분리 규칙]:** 
     - **해결 시:** `[x] Blocker 해결: 누락 수식 판독 완료`
     - **실패 시:** `[x] Blocker 처리 종료: 2회 시도 실패, 판독 불가 태그 삽입`
     - **다음 단계:** `[ ] 본문 작성: 판독된 내용(또는 판독 불가 태그)을 포함하여 마크다운 변환`
