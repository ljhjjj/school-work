# Input_Guide

## 1. 파일 유형 및 분류
- `.txt`: 강의 영상 자막 추출 데이터(STT)
- `.pdf`: PPT 또는 노트 필기 스캔본
- `.jpg` / `.png`: 트러블슈팅 캡처본

## 2. AI 행동 지침 (Data Processing & Workflow)
**[작업 범위 제한]: 새로운 강의 노트 생성/수정 작업에서 파일을 입력받거나 작업을 시작할 때** 다음 지침에 따라 처리하세요. (단순 질문 시 무시)

1. **정보 읽기 및 우선순위 병합 (Preprocessing)**
   - **[노트 필기 PDF] > [PPT 형태 PDF] > [자막 추출 TXT]** 순으로 가중치를 부여합니다.
   - **[.txt 처리: 신뢰도 기반 3단계 STT 교정 원칙]:**
     - **[Tier 1] 확정:** PDF, STT, 문맥이 모두 일치하거나 표준 용어로서 대체 후보가 사실상 하나뿐인 경우. 👉 *본문 즉시 반영.*
     - **[Tier 2] 추정:** 문맥상 가능성이 높으나, 대체 가능한 유사 후보가 2개 이상인 경우. 👉 *가장 적합한 단어로 작성하되 최종 리포트에 추정 내역을 기록.*
     - **[Tier 3] 보류:** 판독 불가. 👉 *원문 유지 후 `[?]` 표시하고 이슈로 기록.*
   - **[.pdf 처리: 명확한 문서 유형 판별 기준]:**
     - **[PPT 형태]:** 슬라이드 번호 존재, 적은 텍스트량, 불릿/명사형 위주, 다수의 도해. 👉 *목차 뼈대 및 키워드 추출.*
     - **[노트 필기]:** 높은 텍스트 밀도, 서술형 문장, 상세한 수식 전개 과정, 수기 필기 흔적. 👉 *부연 설명과 도출 과정 상세 보존.*
   - **[손글씨 수식 중심 노트 처리 규칙]:**
     - 전체 OCR을 수행하지 않는다.
     - 수식은 "유도 흐름 단위"로 해석하여 재작성한다.
     - 원문을 그대로 복사하지 않고, 구조를 재구성한다.
     - 그림은 반드시 설명 텍스트로 변환한다.
     - STT 또는 강의 텍스트가 있으면 반드시 병합한다.

2. **DOC_PLAN 설계 및 작성 (Planning)**
   - 본문 작성 전, **해당 차시의 계획 파일(`[과목명]/[과목명]_[주차]-[차시]_PLAN.md`)**에 단계별 작업 계획을 먼저 작성하세요.

3. **이미지 파일(`.jpg`, `.png`) 트러블슈팅 처리 규칙 (Resolving Blockers)**
   - 타겟 PLAN 파일의 `### ⚠️ 트러블슈팅/이슈` 섹션을 읽고 문제를 판독합니다.
   - **[재요청 제한 및 강제 진행]:** 판독 불가 시 재요청은 **최대 2회**까지만 허용됩니다. PLAN 파일에 재요청 횟수를 명시하세요 (예: `- *Retry Count:* 1/2`)
   - **[상태 업데이트 분리 규칙]:** 
     - **해결 시:** `[x] Blocker 해결: 누락 수식 판독 완료`
     - **실패 시:** `[x] Blocker 처리 종료: 2회 시도 실패, 판독 불가 태그 삽입`
     - **다음 단계:** `[ ] 본문 작성: 판독된 내용(또는 판독 불가 태그)을 포함하여 마크다운 변환`
