# Input_Guide

## 1. 파일 유형 및 분류
프로젝트 내 입력 파일은 확장자와 형태에 따라 다음과 같이 분류됩니다.
- `.txt`: 강의 영상에서 자막 추출 프로그램(STT)을 통해 자동 생성된 로우 데이터 (오타 및 끊김 포함)
- `.pdf`: 개조식 요약 위주의 PPT 형태 강의 자료 또는 줄글/상세 설명 위주의 노트 필기 스캔본
- `.jpg` / `.png`: 트러블슈팅 및 데이터 누락 이슈를 해결하기 위해 사용자가 추가로 제공한 선명한 캡처본(수식, 그래프, 필기 등)

## 2. AI 행동 지침 (Data Processing & Workflow)
새로운 파일을 입력받거나 작업을 시작할 때 다음 지침에 따라 처리하세요.

1. **정보 읽기 및 우선순위 병합 (Preprocessing)**
   - 입력된 `.txt`와 `.pdf` 파일들을 모두 읽고 내용을 종합할 때, **[노트 필기 PDF] > [PPT 형태 PDF] > [자막 추출 TXT]** 순으로 신뢰도와 가중치를 부여하여 정보를 병합하세요.
   - **.txt 처리 (STT 교정 가드레일):** 화자의 발음이나 추출기 한계로 인한 오타 및 스펠링 오류를 교정할 때, **AI의 사전 지식으로 임의의 단어를 추측하여 소설을 쓰지 마세요.** 반드시 함께 제공된 `.pdf`(노트 필기/PPT) 자료에 등장하는 고유 명사, 전문 용어, 교수님 특유의 약어 풀(Pool) 내에서 교차 검증하여 일치하는 단어로만 엄격하게 수정해야 합니다. 불규칙하게 끊긴 문장이나 줄바꿈은 논리적인 하나의 문장으로 이어서 처리하고, 의미 없는 추임새나 스크립트 메타데이터는 제거하세요.
   - **.pdf 처리:** 텍스트의 구조와 밀도를 분석하여 핵심 요약본(PPT)인지 상세 설명본(노트 필기)인지 스스로 판단하고 가중치를 적용하세요.

2. **DOC_PLAN 설계 및 작성 (Planning)**
   - 병합된 정보를 바탕으로 전체 마크다운 문서의 대주제(H2)와 소주제(H3) 목차 구조를 설계하세요.
   - 본문 작성을 시작하기 전에, 설계한 목차와 단계별 작업 계획을 `DOC_PLAN.md`에 포맷에 맞춰 가장 먼저 작성하세요.

3. **이미지 파일(`.jpg`, `.png`) 트러블슈팅 처리 규칙 (Resolving Blockers)**
   - 사용자가 이미지 파일(`.jpg`, `.png`)을 제공한 경우, 즉시 `DOC_PLAN.md`의 `### ⚠️ 트러블슈팅/이슈` 섹션을 읽고 현재 대기 중인(Blocked) 문제가 무엇인지 파악하세요.
   - 제공된 캡처본에서 누락되었던 수식, 텍스트, 훼손된 문맥을 시각적으로 판독하여 추출하세요.
   - 에러가 해결되면 즉시 `DOC_PLAN.md`의 이슈 내용을 지우고 해당 파트의 체크박스를 완료(`[x]`) 처리한 뒤, 멈췄던 부분부터 마크다운 본문 작성을 즉각 재개하세요.
