# Web Cross-Validation Guide

## 1. 목적
로컬 Windows 환경의 Codex가 인터넷 검색을 통해 강의 노트의 전문 용어 및 공식을 제한적으로 교차 검증하는 기준과 절차를 정의한다. 인터넷 자료는 본문의 새로운 근거가 아닌 **단순 검증 보조 자료**로만 활용한다.

## 2. 실행 시점
이 가이드는 다음 경우에만 참조한다.
- 마크다운 본문 초안 작성이 끝난 뒤 최종 검수 단계에 진입했을 때
- STT 오류가 의심되는 전문 용어가 남아 있을 때
- 보편 공식, 단위, 수학/물리 기호의 표기가 의심될 때
- PLAN 또는 본문에 `[?]`, `추정`, `확인 필요` 태그가 남아 있을 때
- `validation_guide.md` 검증 이후, 최종 리포트 작성 이전에 수행한다.

단, 단순 질문, 문장 다듬기, 일반 요약 요청에는 이 가이드를 발동하지 않는다.

## 3. 사전 환경 설정 (Network Access)
- Windows 로컬 Codex의 기본 `workspace-write` 샌드박스는 네트워크 접근이 차단되어 있을 수 있다.
- 필요시 `~/.codex/config.toml` 등의 설정에서 네트워크 접근(`network_access = true`)이 허용되어 있는지 확인하거나 전제한다.
- **[경고]** 인터넷 접근이 가능하더라도 `codex_policy.json`에 명시된 대로 `npm install`, `pip install` 등 파괴적이거나 시스템 변경을 가하는 외부 패키지/모듈의 다운로드 및 설치는 명시적 승인 없이 절대 금지된다.

## 4. 제한적 검색 대상
전체 내용을 인터넷 자료를 기준으로 재작성하는 것을 엄격히 금지하며, 아래 항목에 한해서만 웹 검색을 수행한다.
1. STT(음성 인식)에서 오류가 의심되거나 불분명하게 들린 전문 용어 확인
2. 보편적인 공식, 단위, 수학/물리 기호의 오타 및 누락 확인
3. 영문/한글 표준 전공 용어 표기법 교차 검증

## 5. 정보 반영 우선순위
- **원본 절대 우위 원칙:** `노트 필기 PDF > PPT PDF > STT TXT > 인터넷 자료`
- 원본 강의 자료(교수님 자료)와 인터넷 검색 결과가 충돌할 경우, **반드시 원본 강의 자료의 표기와 논리를 우선**한다.

## 6. 금지사항
- 인터넷 검색 결과를 근거로 강의자료의 전개 방식, 교수자 표기, 예제 순서를 임의 변경하지 않는다.
- 블로그, Q&A, 위키성 자료 하나만 보고 본문을 확정하지 않는다.
- 대량 크롤링, 파일 다운로드, 외부 스크립트 실행, 패키지 설치를 수행하지 않는다.
- 검색 결과가 불확실하면 본문을 단정적으로 수정하지 않고 `추정` 또는 `확인 필요`로 기록한다.

## 7. 기록 형식
웹 교차 검증을 통해 본문을 수정하거나 단어를 추정한 경우, 작업 완료 후 최종 리포트 및 타겟 PLAN 파일에 **"외부 검증 기반 수정/추정 내역"**으로 반드시 명시하여 사용자가 인지할 수 있도록 한다.

### PLAN 기록 예시
#### 외부 검증 기반 수정/추정 내역
- `모멘트 암`으로 들린 STT를 문맥 및 표준 용어 기준으로 `moment arm`으로 확인하여 반영함.
- `프와송 비` 표기를 원본 PDF와 표준 용어가 일치하여 `포아송 비(Poisson's ratio)`로 정리함.
- `전단계수 G` 공식은 원본 필기와 표준 공식이 일치하여 확정 처리함.

### 최종 리포트 기록 예시
- 외부 검증 기반 수정: 3건
- 외부 검증 기반 추정: 1건
- 원본 우선으로 외부 자료를 반영하지 않은 항목: 1건
